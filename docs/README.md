- タスク・コミット・プルリク・リリースは粒度を小さくできます

- 簡単な英会話であればできます

## プロダクトに関する考え方

- ユーザーファースト
- ユーザーの要望をそのまま機能にするのではなく、機能が必要になった背景を把握したうえで、機能開発をしたい
- ユーザーに早く機能を届け、高速なフィードバックを受けながら、より良い機能を提供していきたい
- ユーザーが使う機能をユーザー、データに基づいて開発をしたい
- QCD（Quality、Cost、Delivery）を意識して開発したい
